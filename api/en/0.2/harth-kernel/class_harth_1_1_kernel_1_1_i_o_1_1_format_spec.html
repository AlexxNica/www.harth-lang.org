Parse printf() like FormatSpec flags, padding, width, and precision.

Possible output flags: 0 - pad left with zeroes

  • - place sign always before number (otherwise only - appears). <sp> - place a space for positive values, - for negative values.
  • - pad on right with blanks (0 flag ignored) ^ - pad on center (odd blank left, - means odd blank right)

    - add base prefix (nothing for decimal), 0 for octal (unless

already starts with zero), 0x for hex @ - pad with selected byte, means pad with char 'x' @* means character to pad is an argument, @ means next character is literal (ie. for padding with '*' and '@' use '@*' and '@@').

[0-9]+ - width of elements/characters, typically minimum number of digits/characters .[0-9]+ - precision of elements/characters, typically maximum number of digits/characters ![0-9]+ - precision set to current WriteFormat depth with given value added, useful for limiting output to certain depths (eg. '%!1F' means output contents of a list but with elements as forwards).
